API Overview

The CryptoHub API provides developers with programmatic access to cryptocurrency market data, trading capabilities, and blockchain analytics. Our RESTful API uses standard HTTP methods and response codes, returning JSON-encoded responses.

All API endpoints are served over HTTPS. The base URL for all endpoints is:

https://api.cryptohub.com/v1

Authentication

To access the CryptoHub API, you need an API key which you can generate from your account dashboard. API keys are passed in the request header.

GET
/account
Retrieve account information including balances and permissions.
Headers
X-API-KEY
string
Your API key
cURL
JavaScript
Python
curl -X GET "https://api.cryptohub.com/v1/account" \
  -H "X-API-KEY: YOUR_API_KEY"
fetch('https://api.cryptohub.com/v1/account', {
  method: 'GET',
  headers: {
    'X-API-KEY': 'YOUR_API_KEY'
  }
})
.then(response => response.json())
.then(data => console.log(data));
import requests

url = "https://api.cryptohub.com/v1/account"
headers = {"X-API-KEY": "YOUR_API_KEY"}

response = requests.get(url, headers=headers)
print(response.json())

Rate Limits

To ensure fair usage and system stability, CryptoHub API enforces rate limits:

  • Free Tier: 100 requests per minute
  • Basic Tier: 300 requests per minute
  • Pro Tier: 1,000 requests per minute
  • Enterprise: Custom limits

Rate limit headers are included in every response:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1634295600

If you exceed your rate limit, you will receive a 429 Too Many Requests error.
